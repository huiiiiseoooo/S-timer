## 📝 Git Commit 컨벤션 (규칙)

우리 프로젝트는 일관된 커밋 히스토리 관리와 원활한 협업을 위해 아래의 커밋 메시지 구조와 타입을 철저히 따릅니다.

### 🏷️  커밋 타입 (Type) 종류
커밋 작성 시 반드시 아래의 타입 중 하나를 선택하여 맨 앞에 작성합니다.

- **feat** : 새로운 기능 추가
- **fix** : 버그 수정
- **docs** : 문서 내용 수정 (README 등)
- **style** : 코드 포맷팅, 세미콜론 누락 등 (비즈니스 로직 변경 없음)
- **refactor** : 코드 리팩토링 (기능 변화 없음)
- **test** : 테스트 코드 추가 및 수정
- **chore** : 빌드 업무, 패키지 매니저 설정, 의존성 추가 등
